Validation des fonctions des applications_

En informatique, un test désigne une procédure de vérification partielle d'un système. Son objectif principal est d'identifier un nombre maximum de comportements problématiques du logiciel afin d'en augmenter la qualité. Un 'bug / bogue' peut avoir des conséquences désastreuses s'il n'est pas corrigé avant l'utilisation du logiciel.
Prenons un exemple afin de bien comprendre leurs conséquences :

Ariane V

Un morceau de code d'Ariane IV ( concernant le positionnement et la vitesse de la fusée) repris dans Ariane V.
Il contenait une conversion d'un flottant sur 64 bits en un entier signé sur 16 bits.
Pour Ariane V, la valeur du flottant dépassait la valeur maximale pouvant ête convertie.
Défaillance dans le système de positionnement.
La fusée a "corrigé" sa trajactoire.
Suite a une trop grande déviation, Ariane V est détruite.
Coût du bogue ?
Pas moins de 5 000 milliards de dollars

Quel impact ?
* Sécurité des personnes
* Relations contractuelles
* Notoriété, image, …

Il est donc nécessaire de « vérifier » certains logiciels/systèmes.

Différents types de tests_

Afin d'assurer la vérification des systèmes et logiciels, pluieurs types de tests sont apparus:
Des tests unitaires
Ils permettent de vérifier le bon fonctionnement d'une partie précise d'un logiciel. Ils ne peuvent pas être automatisés.
Des tests d'intégrations
Ils vérifient que chaque modules indépendants du logiciel est assemblé et testé dans l’ensemble.
Des tests de validations
Ils vérifient si toutes les exigences client, décrites dans le document de spécification du logiciel, sont respectées.
Des tests de non-régression
Ils vérifient que l’ajout ou la modification de fonctionnalités existantes ne menacent pas le bon fonctionnement de l’application.
Il est également possible d’automatiser des tests de non-régression qui seront joués à volonté à la suite de livraison d’une nouvelle version de l’application :

L'automatisation d'un test n'a de sens que si le test répond à un certain nombre de critères :
Le test est systématique : il doit être exécuté à chaque nouvelle version de l'application.
Le test est répétitif : il est présent dans de nombreux scénarios de test.
Le test est automatisable : il est possible techniquement de faire jouer le test par un robot.
